# Frontend Migration Guide: Backend v2 (Gemini API Upgrade)

This document outlines the changes made to the Backend `v2` routes. All `v2` endpoints have been migrated from local Mistral models to the **Gemini 2.5 Flash API**.

## 🚀 Overview of Changes
- **Performance**: Response times are significantly faster.
- **Accuracy**: Improved reasoning and retrieval quality.
- **Resource Usage**: Lower local server RAM consumption (local model weights are no longer loaded).

## 🛠 Required Frontend Adjustments

### 1. Flashcards with Citation
*   **Endpoint**: `/flashcardwithcitation/generate-flashcards/`
*   **Change**: The "Source" field has been upgraded from a line number to actual text.

| Key | Old Value (Mistral) | New Value (Gemini) | Action |
| :--- | :--- | :--- | :--- |
| `source_text` | N/A | `"Actual quote from the PDF..."` | **Display this text** as the citation. |
| `source_line` | `12` | (Deprecated/Removed) | Remove logic relying on numeric line numbers. |

**Example New Response Structure:**
```json
{
  "model": "Gemini-2.5-Flash",
  "total_flashcards": 5,
  "flashcards": [
    {
      "question": "What is the primary goal of RAG?",
      "answer": "To ground LLM responses in verifiable source data.",
      "difficulty": "Medium",
      "source_text": "Retrieval-Augmented Generation (RAG) aims to reduce hallucinations by providing external context."
    }
  ]
}
```

---

### 2. General v2 Endpoints (Chat, Summary, Quiz, MCQ)
These endpoints are **mostly backward compatible**. All JSON keys remain the same, but you can now expect a new metadata field in the response.

*   **New Field**: `engine` or `model` will now return `"Gemini API"` or `"Gemini-2.5-Flash"`.
*   **Context Volume**: The backend logic now sends larger document chunks to the LLM. The frontend doesn't need to change anything to benefit from this!

---

### 3. Concept Graphs (Insight Generation)
This route remains **Hybrid**.
*   It accepts a `use_api` Boolean flag.
*   If your UI has a "Fast/API Mode" toggle, send `use_api: true` (recommended).
